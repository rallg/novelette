What is Novelette?

Novelette is a document class, with support files, created as a "package" for TeX typesetting software. It is new in TeX Live 2024.

The master development site is "github.com/rallg/novelette".

What does it do?

Novelette does only one thing: Printed, original popular fiction (detective novels, historical romance, and the like). It is structured for the needs of print-on-demand, with interior pages using only black ink.

You may include raster images. You may create a color cover, if you have additional graphics support.

What does it not do?

Available commands and features are very limited, for the intended usage. Novelette does not support Ebooks, mathematics, graphic novels, or academics. You may not include vector drawings, or other PDF files.

This distinguishes Novelette from nearly all other LaTeX packages. Requests for new features are strongly discouraged.

For best typesetting, you will need to edit and rewrite parts of your book, as you go along. This is why Novelette is intended for original fiction, where you are free to rewrite anything. If you are trying to reprint someone else's work, then you should use something other than Novelette.

What does it need?

You need a working LaTeX installation, with LuaTeX support, and a variety of other packages. Even a small TeX setup is likely to have most or all of it. Anything missing can be added via your TeX package manager.

  • If your book has images, or if you wish to use Novelette to process a cover, then you will need the GraphicsMagick program. This is not part of TeX, but it is free for Linux, Windows, and Mac.
  • You will need a PDF reader, to view the result of your work. If you are using Linux, the Evince (document viewer) program has the capability to update an opened PDF when you edit the TeX document. The two-window TeXworks program also has this capability, on all platforms. So does some other software.
  • No matter what you use to view the PDF as you write, view the final result on Windows or Mac using Adobe Acrobat Reader (the free version is enough). Reason: Adobe Acrobat is likely to be the software used for screening the PDF, when it it is submitted for printing. If you do not have Windows or Mac, or no Adobe Reader, find a friend or library that has it.

How much Lua do I need to know?

None. The Lua code is all invisible to you.

How much LaTeX do I need to know?

The less, the better. Novelette does not implement many standard LaTeX commands. This is intentional. If you have already used TeX for academic work, then you will need to stop doing whatever you did for your research papers.

Novelette has a small set of commands, targeted to its intended purpose. Although TeX experts can add more, popular fiction has little need of more.

What languages does it support?

English, French, German, Italian, Catalan, Spanish. Some have options. That's it.

How can I check my setup?

First, be sure that you have a working LuaLaTeX installation with Novelette installed. Compile this test document. You may load it into TeXworks (compiler will be lualatex) or use lualatex test on command line. Copy and save as file test.tex :

% !TeX TS-program = lualatex
% !TeX encoding = UTF-8
\documentclass{novelette}
\begin{document}
Hello, World.
\end{document}

If that works, then you are ready. But if that did not work, then you will not be able to proceed without fixing it.

Although it is not always necessary, you will get additional capability if you also install package libertinus-fonts using your TeX package manager.

How to write?

You may use a plain-text editor, or word processor.

  • Advantage of a word processor is spelling check, automated smart quotes, and (often) a menu for choosing special characters (with diacriticals).
  • Novelette does not understand mathematical symbols, so avoid using a TeX GUI program that is intended for mathematics.
  • In all cases, the *.tex documents must be plain text, encoded UTF-8, without BOM (byte order mark). Nowadays, this is the default plain text exported by most programs. Be sure that a word processor did not insert any of its own program codes.

Where do I start?

Ready to write? Then start with the Overview and continue through the documentation.

If you have used LaTeX before, or have been reading the online documentation for other packages, you may have seen many commands that Novelette does not use. So before decorating your story with numerous commands that will not work, see if the ones you want are listed in the Index.
