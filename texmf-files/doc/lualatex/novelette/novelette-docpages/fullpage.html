Full-Page Environments

Terminology

A full-page environment stands by itself. It begins a new page. When the environment ends, the following material begins a new page. Although a full-page environment is normally just one page, in some cases it may extend to a second page, which is also full-page.

Do not confuse full-page environments with floating full-page images. You cannot use any full-page environment in the middle of a chapter (because it will break the chapter into separate pieces).

There are three distinct full-page environments: fullpage copyrightpage, and contents. The first of these is generic, without any special style or format. The others are specialized, for their indicated purpose.

\begin{fullpage}...\end{fullpage}

This is the generic full-page environment. It is suitable for such things as half-title page, title page, maps (if not mid-chapter), and some other purposes where there is no particular format.

If page layout has a header, it will be empty. If page layout has a footer, it will be empty in frontmatter, but print the page number in mainmatter. This behavior may be changed using \thispagestyle within the environment.

Within this environment, you may use nearly any style command, as well as images and icons.

You may place an upperpage environment at the top of fullpage. This is useful if the lower portion of the page has ordinary text, and you wish that text to be on-grid.

If you place an image in fullpage, it cannot float. It will appear where you write the \image command.

If fullpage does not fit on a single page, it will extend to another page (automatically fullpage). An alert is issued, because you might not have intended more than one page.

\begin{copyrightpage}...\end{copyrightpage}

You do not need to use this environment, if you prefer to style the copyright page yourself. But it is recommended, because it pre-styles its contents, and thus involves less coding on your part.

The copyrightpage does not print header or footer. Its text is horizontally centered. Paragraphs are not indented, and are separated by a blank line. Hyphenation is disabled. Numerals are lining, not oldstyle. You may not use upperpage, \name, \subname, or any of the \formatX commands. You may not use \image (but you may use \icon). You may use Italics, small caps, and \smalltext.

If the content of copyrightpage does not fit on one page, it is an error.

Be sure you understand what information is placed (or not placed) on a copyright page, regardless of how you format it.

\begin{contents}...\end{contents}

For ordinary novels, a table of contents is unnecessary and discouraged. Readers will not need it. But if your book is a collection of short stories, then a table of contents may be useful.

Nothing within contents is automated. The page title, whether it be CONTENTS, or Table of Contents, or Table de Contenu, or something else, is for you to provide. Use name, or \subname, or one of the \formatX commands, to provide its style. But it will be force-aligned to center.

Use \entry{chapter}{page} as needed. This is not automated. Do it yourself!

At the top of contents you may write \linegap{N}, where N is a number >=0 and <=1. This increases line-to-line spacing. It is useful if the table is short, or it it is so long that it continues to a second page.

If page layout uses header, it will be empty. If page layout uses footer, it will be empty. Some books put a page number on the contents page, but Novelette does not.

Here is a sample contents page:

\begin{contents}
\linegap{.25} % spread them
\null\null % space at top
\subname{CONTENTS} % or some other style
\null
\entry{1. The Widget}{1}
\entry{2. Free Beer}{14}
\entry{3. Too Good For You}{35}
\entry{4. A Pleasant Surprise}{54}
\entry{5. Aghast in Paris}{112}
\entry{6. Beyond Belief}{187}
\end{contents}

\formatX{text} (where X is a, b, c, d, e, f)

Commands \formata{text through \formatf{text} may only be used in the fullpage environment. Each of these commands styles its text according to the corresponding \formatXstyle setting in Premable. These commands are useful for such things as title page, where the various lines of text have different sizes, alignments, and sometimes fonts.
