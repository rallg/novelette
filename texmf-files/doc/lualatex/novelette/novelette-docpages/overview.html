Folder Organization

For best results, and ease of editing, organize your book into several folders (directories) and files. All of this documentation assumes you will do that.

All folder names, and all file names: no space allowed; no underscore allowed; no numbersign (#) allowed, no ampersand (&) allowed, no tilde (~) allowed. There are other disallowed characters, but they would not normally be in a folder name or file name.

Create a folder that will contain all of your work: text, images, notes to yourself, and so forth. Then, you can copy or transfer that folder to another device, or zip it to an archive, without losing anything.

Within that folder, create subfolders that will contain the various portions of your book. Also create a subfolder that will contain items that will not appear in the book, such as alternative plots, backup files, native-format image files, reference information, and the like.

A typical structure might look like this, for the interior-demo included with this documentation:

novelette-demo % top-level folder
|-- novelette-demo-interior.tex % main document
|-- A. frontmatter % folder
|-- |-- frontmatter-i-ii.tex % pages i-ii
|-- |-- frontmatter-iii-vi.tex % pages iii-vi
|-- B. mainmatter % folder
|-- |-- lying % folder
|-- |-- art-of-lying.tex
|-- |-- cooper % folder
|-- |-- literary.tex
|-- |-- river % folder
|-- |-- mississippi.tex.tex
|-- |-- scenes % folder
|-- |-- |-- miss-1.tex

|-- C. backmatter % folder

Using Subfiles

Novelette has built-in support for subfiles. This allows you to divide your document into small files (such as chapters) that may be compiled alone, for testing and editing. The above folder structure shows how subfiles are used.

Here is how a typical subfile is coded:

% !TeX TS-program = lualatex
% !TeX encoding = UTF-8
\documentclass[../../conspiracy-manifesto.tex]{subfiles}
\begin{document}
Content of this chapter.
\end{document}

In the above sample code:

  • The document class is subfiles instead of novelette.
  • The class option is the path to the main document. In this example, the chapter is two folder levels below the main document, so ../../ is prefixed to the main document filename conspiracy-manifesto.tex with its file extension.
  • If you compile the subfile by itself, the PDF metadata title will be changed, so that it is not confused with the entire book. Printed titles are unchanged.
  • Any settings in the main document Preamble will be applied to the subfile.
  • You may write \mode{setting} and \guides{setting} in the subfile Preamble. They will only be used when compiling the subfile by itself. They will be ignored when compiling the main document.
  • You cannot compile a single subfile in final mode. It will automatically revert to draft.
  • If a subfile has images or icons, their paths are relative to the main document, not relative to the subfile. It is helpful to put all images and icons into a single folder, so that the path is always the same.
  • You may call other subfiles from within a subfile. This is useful if you have (for example) a long chapter (subfile of main) with several scenes (subfiles of the chapter).
    • The path to sub-subfiles is relative to the calling subfile. So in the cas of a chapter (subfile of main document) with scenes (subfiles of the chapter), the path to the scenes is relative to the chapter.
    • It may be helpful to place a chapter's upperpage environment as a subfile, which then calls its text as sub-subfiles. If there are scene labels, put them in the chapter file. With this strategy, there will be very little (or no) LaTeX code in the sub-subfiles. That makes it much easier to transfer the files, as plain text with little or no coding, to another person for editing.
  • The cover is not a subfile of the main document. It stands by itself. When you process a cover *.tex file, the path to the cover image is relative to this file. Best to put it in the cover folder, not in the images folder.
