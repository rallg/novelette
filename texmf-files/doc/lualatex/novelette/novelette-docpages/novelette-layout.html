Book Size and Page Layout

The width and height of book interior pages is its trim size, because larger paper is fed into the printing machine, then trimmed (cut) to final size during binding.

Always use off-white (cream, creme) paper for popular fiction. In the USA, popular fiction often uses trim size 5.5in width, 8.5in height. In the UK, perhaps a popular size is B-format, at 129mm width, 198mm height. The choice is yours. Look at other books in your genre.

Book Size

The Novelette term for trim size is book size, to avoid confusion with similar terms. It is set using the \booksize command in Preamble. You must set it. There is no default size. Failure to set it will produce an immediate fatal error.

  • Command: \booksize{w=width,h=height}
  • Width and height have units of in (inches) or mm (millimeters).
  • Height may not be less than width.
  • Novelette does not check the size against a standard list. Unless the setting is absurdly small or large, what you set is what you get.

^return

Sheet Feed

Do not use this command unless necessary.

Novelette normally produces PDF page size that is equal to the booksize (trim size). This is how nearly all print services want it.

If you are printing a copy on your home printer, perhaps for manual editing, most printers will have no problem placing the PDF pages on whatever paper you feed. But in the event that your printer does not know what to do, then sheetfeed may help. Be sure to remove this command, before the final PDF.

In rare instances, your print service may ask you to do this, usually if there is a human editor. Do not do it unless explicity requested.

Command: \sheetfeed{paper} % paper is letter or A4.

^return

Margins

You do not need to set margins, since Novelette has defaults. But if you wish to do it yourself...

In Novelette, the four margins are boundaries of a rectangle. This rectangle is the printable area. All content (including header, footer, main text, and images) must fit within the printable area.

Some word processors do not define "margins" that way. Instead, they will place the header within the area of the top margin, and place the footer within the area of the bottom margin. If the print service suggests margins, be sure that the values account for the way Novelette does it, not the way a word processor does it.

All print dervices require some minimum margin values. You cannot print anything too close to the page edge or the binding, because print-on-demand has more manufacturing variation than most other methods. You cannot use interior bleed (images trimmed at page edge).

As far as the Novelette developer knows, margins of 13mm at top, outside, and bottom, and 16.2mm at inside, will meet the minimum margins widths of print-on-demand services in the USA and Europe. These are the defaults. If you are allowed to use narrower margins, or if you wish to use wider margins, then set them yourself in Preamble.

In general, the inside margin is wider than the outside margin, by about 3mm or 0.125in, to allow for glue at the spine binding. There is no separate setting for "gutter".

  • Command: \margins{t=top,o=outside,b=bottom,i=inside}
  • Each margin has units of in (inches) or mm (millimeters).
  • If you set any margin, then you must set all four of them.
  • The printable content should look like it is centered in the page. Artistic effects, such as an extra-wide outside or bottom margin, may be rejected by the print service. This is because the humans who handle the pages have never seen your book before, and may think that the printer was mis-fed, if the pages are visibly off-center.

Type Size (Main Font Size)

Novelette uses the term typesize to mean the size of the main body text, measured in units of pt (TeX points), or bp (PostScript points).

You do not need to set typesize, since Novelette will guess a value (11 to 12 pt, usually in between) based on various other settings. It is often best to accept the guessed value. But if you wish to do it yourself...

  • Command: \typesize{size}
  • The size has units of pt or bp.
    • This is a Preamble setting. You cannot change it later in the book. However, larger or smaller text will be used in special locations.

  • You cannot set the leading, known to TeX as baselineskip. It will automatically be calculated from other settings.

^return

Lines Per Page (Main Text Block)

You do not need to set the number of lines per page. Novelette will calculate a good value, based on other settings. But if you wish to do it yourself...

  • Command: \lines{integer}
  • The integer is lines per page in the main text block, not counting header or footer.
  • This is a Preamble setting. You cannot change it later in the book.

^return

Header and/or Footer Layout

Although Novelette has a default layout, it is not always best. So be sure to read this, and choose a layout that works for you.

Novelette does not allow you to construct your own layout. You must choose from several pre-defined styles. Just about every book, in the popular fiction of public libraries, will have one of these styles. Yes, the developer did look through the shelves.

  • Command: \layout{style=S,hgap=H,fgap=F,num=N}
  • S is an integer 1-7. It chooses whether layout has header and/or footer, and alignment of content. The choices are described and illustrated in the gallery of layout styles. Default style is 2.
  • H is a number from 0 to 1 (usually in between). It sets the extra vertical space between header and main text, measured in lines. Value 0 is usually not practical. Value 1 inserts a full empty line. Default value is 0.5, which is reasonable for many books.
  • F is like hgap, except it sets space between main text block and footer. Default 0.5.
  • N is a one-letter or two-letter code, chosen from these possibilities: m l o i mi li oi. If the code contains i, then page numbers (roman or arabic) will be italicized. If the code contains one of m l o, then arabic numbers will use the same style as main text (m), or lining numerals (l), or oldstyle numerals (o). This assumes that your choice is actually available in the font. Default is m.

^return

Guides

Guides can assist with visual alignment, on pages that use white space, such as chapter openings. You can see an example.

Guides do not force anything. They do not affect the layout. How you set guides, and whether or not you obey them, is up to you. There are no default guides.

  • Command: \guides{up to four comma-separated integers}
  • The integers are number of main text lines, from top of main text.
  • Example: \guides{8,12,15,22} sets four guide pairs.
  • This is a Preamble setting. It cannot be changed later in the book.
  • Guides may be set regardless of mode. But they are only visible when in shade mode.
  • The bottom edge of guides is at main text baselines. The top edge is as the normal x-height.

^return
