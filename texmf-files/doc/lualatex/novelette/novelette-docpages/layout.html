Book Trim Size and Page Layout

This page is under development.

The width and height of book interior pages is its trim size. Larger paper is fed into the printing machine, then trimmed (cut) to final size during binding.

Always use off-white (cream, creme) paper for popular fiction. Choose a trim size from among the standards offered by the print service. Beware: The most commonly-used sizes for commercial publishing (sold in large quantity to libraries and stocked on bookseller shelves) may or may not be the best sizes for print-on-demand softcover fiction.

Trim Size

The trim size is the width and height of book interior pages. This is an industry standard term. Novelette does not provide a default setting. If you do not set it yourself, an immediate fatal error results.

  • Command: \trimsize{w=width,h=height}
  • Width and height have units of in (inches) or mm (millimeters).
  • Height may not be less than width.
  • Novelette does not check the size against a standard list. Unless the setting is absurdly small or large, what you set is what you get.

Preamble:\trimsize top

Sheet Feed

Do not use this command unless necessary.

Novelette normally produces PDF page size that is equal to the trim size. This is how nearly all print services want it.

If you are printing a copy on your home printer, perhaps for manual editing, most printers will have no problem placing the PDF pages on whatever paper you feed. But in the event that your printer does not know what to do, then sheetfeed may help. Be sure to remove this command, before the final PDF.

In rare instances, your print service may ask you to do this, perhaps if there is a human editor. Do not do it unless explicity requested.

  • Command: \sheetfeed{paper}
  • The paper is letter or A4.

Margins

In Novelette, the four margins are boundaries of a rectangle. This rectangle is the printable area. All content (including header, footer, main text, and images) must fit within the printable area.

This illustrates an open book spread, for a page layout that has both header and footer. Printable area has white background. Non-printable margins are gray. Note that each inside margin is slightly wider than the outside margins:

 bottom margin   bottom margin 

Some word processors do not define "margins" the same way. Instead, they place the header within the area of the top margin, and place the footer within the area of the bottom margin. If the print service suggests margins, be sure that the values account for the way Novelette does it, not the way a word processor does it.

  • Command: \margins{t=top,o=outside,b=bottom,i=inside}
  • Example: \margins{t=.5in,o=.5in,b=.5in,i=.625in}
  • Each margin has units of in (inches) or mm (millimeters).
  • If you set any margin, then you must set all four of them.
  • Inside margin is generally wider than outside margin, because inside margin includes the glue binding area. Typical extra width is 3mm or 0.125in.
  • Novelette does not allow page bleed. All margins must have a value greater then zero.
  • Print services require minimum margins. The defaults are 13mm top, outside, and bottom; 16.2mm inside. This appears to meet the requirements of all known print-on-demand services. Some allow smaller values.
  • Artistic effects, such as an extra-wide outside or bottom margin, may be rejected by the print service. This is because the humans who handle the pages have never seen your book before, and may think that the printer was mis-fed.

Type Size (Main Font Size)

Novelette uses the term typesize to mean the size of the main body text, measured in units of pt (TeX points), or bp (PostScript points).

You do not need to set typesize, since Novelette will guess a value (11 to 12 pt, usually in between) based on various other settings. It is often best to accept the guessed value. But if you wish to do it yourself...

  • Command: \typesize{size}
  • The size has units of pt or bp.
  • This is a Preamble setting. You cannot change it later in the book. However, larger or smaller text will be used in special locations.

  • You cannot set the leading, known to TeX as baselineskip. It will automatically be calculated from other settings.

Preamble:\typesize top

Lines Per Page (Main Text Block)

You do not need to set the number of lines per page. Novelette will calculate a good value, based on other settings. But if you wish to do it yourself...

  • Command: \lines{integer}
  • The integer is lines per page in the main text block, not counting header or footer.
  • This is a Preamble setting. You cannot change it later in the book.

Header and/or Footer Layout

Although Novelette has a default layout, it is not always best. So be sure to read this, and choose a layout that works for you.

Novelette does not allow you to construct your own layout. You must choose one of several pre-defined schemes. Just about every book, in the popular fiction of public libraries, will have one of these schemes. Yes, the developer did look through the library shelves.

Non-fiction books often have distinctive page headers, because the book will be read slowly. But fiction books normally have simple page headers, because the book will be read quickly. Resist the temptation to create distracting page headers.

  • Command: \layout{scheme=I,hgap=H,fgap=F,font=T}
  • I is an integer 1-7. It chooses whether layout has header and/or footer, and alignment of content. The choices are described and illustrated in the gallery of layout schemes. Default scheme is 2.
  • H is a number from .5 to 1. It sets the extra vertical space between header and main text, measured in lines. Value .5 inserts a half-high empty line. Value 1 inserts a full empty line. You may use values in-between. Default 1.
  • F is like hgap, except it sets space between main text block and footer. Default 1.
  • T is an integer 1-4. Default 1 uses Swainson Regular/Italic, same as main text (but tweaked). Choices 2 and 3 provide more dramatic header/footer. Choice 4 requires package libertinus-fonts installed, and \enable{libertinus} in Preamble.

Guides

Guides can assist with visual alignment, on pages that use white space, such as the upperpage environment. You can see an example.

Guides do not force anything. They do not affect the layout. How you set guides, and whether or not you obey them, is up to you. There are no default guides.

  • Command: \guides{up to four comma-separated integers}
  • The integers are number of main text lines, from top of main text.
  • Example: \guides{8,12,15,22} sets four guide pairs.
  • This is a Preamble setting. It cannot be changed later in the book.
  • Guides may be set regardless of mode. But they are only visible when in shade mode.
  • The bottom edge of guides is at main text baselines. The top edge is as the normal x-height.
