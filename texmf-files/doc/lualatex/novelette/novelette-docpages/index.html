This page describes settings for the book interior pages. For information regarding cover artwork, go here.

Minimal Preamble

Almost everything in Novelette has a pre-defined default setting. You may be able to write your book without learning most commands.

This is the minimal document Preamble, enough to create a minimal working example. The PDF will be in draft mode, which has correct typesetting but is internally marked as draft:

\documentclass{novelette}
\booksize{w=width,h=height} % Required.
\begin{document}
Contents of your book go here.
\end{document}

Complete Preamble

Novelette defines a number of Preamble settings. Most of these settings have reasonable defaults, so you do not need to set all of them.

You must set \booksize. This is the "trim size" of the book interior pages. There is no default.

Note that you cannot use \usepackage or \RequirePackage to add more packages. Novelette is very restrictive, because added packages are likely to break other settings. The packages you need are already included. If you wish to add something else, then either do without it, or use a different document class.

Here is a list of all available Preamble settings, for book interior. Click a setting, and you will navigate to its HTML documentation:

% !TeX TS-program = lualatex
% !TeX encoding = UTF-8
\documentclass{novelette}
\title{Title of Book
\author{Author of Book}
\subtitle{Subtitle of Book}
\mode{setting}
\intent{setting}
\booksize{setting} % Required.
\sheetfeed{setting} % Rarely used.
\margins{setting}
\typesize{setting}
\lines{setting}
\layout{setting}
\guides{setting}
\typeface{setting}[options]
\sansfont{setting}[options]
\monofont{setting}[options]
\altfont{setting}[options]
\dramafont{setting}[options]
\decofont{setting}[options]
\namestyle{setting}
\descstyle{setting}
\setmicrotype{setting}
\myparindent{setting}
\warnpercent{setting}
\warnquotes{setting}
\warnalert{setting}
\application{setting}
\producer{setting}
\begin{document} % End of Preamble.
Commands that may be used in the document body are on another page.
\end{document}

Description of Preamble Settings

Magic Comments

Although they are not required, it is helpful to write these two lines at the very top of your document (including the % symbol):

% !TeX TS-program = lualatex
% !TeX encoding = UTF-8

Those are "magic comments" that are understood by some GUI programs, such as TeXworks. They tell the program how to compile your document.

If you are compiling from command line, the magic comments are ignored.

^return

\documentclass{novelette}

With the exception of cover artwork, there are no document class options. All settings are performed in Preamble using the commands described here. This behavior is very different from standard LaTeX document classes (such as article or memoir).

^return

\title{Book Title} and \author{Author Name}

If you are not producing your document in final mode, then you do not need to provide title or author. Novelette will automatially insert "Untitled Document" and "Anonymous Author", if you do not set them.

In final mode, you must provide both title and author. There are some restrictions:

  • You are limited to the Latin-1 character set. This is partly due to Novelette restrictions, and partly due to limitations of the print-on-demand distribution system.
  • You cannot use curly single or double quotes, or curly apostrophe, or dashes. These are not Latin-1 characters. You may use the straight single and double quotes, and single hyphen. So if your author is "Pat O'Doule", use the straight apostrophe.
  • TeX has numerous special characters, which must be escaped with backslash, if you wish to write them in title or author. Codes \# \$ \% \& \{ \} might be necessary.
  • Although TeX allows you to use the above backslashed characters in title and author, your book distributor may disallow some or all of them. Ask. For example, you may need to spell "Dollar" rather than use "\$" in a title such as "The Thousand-Dollar Ransom". This has to do with the way book titles are indexed.
  • The author may be a pseudonym (pen name). Be sure to use your real name as Copyright Holder, on the Copyright Page, so you can prove that author "Joe Fakename" is "Really You", in case of copyright infringement.
  • In general, print-on-demand services require that the book title and author be identical, anywhere they appear. Whatever it says on the cover artwork, must match the title page, the copyright page, and (if used) the running page headers. If your full name is "Aloysius Lightlyrunningbear Tornithengle", the disributor may allow you to abbreviate it as "A. L. Tornithengle" in places. Ask them.

    If you use a pseudonym, it is a marketing tool. For example, the Novelette developer has a surname that does not harmonize with his writing genre, so he uses a more harmonious pseudonym. Do not imagine that you can hide your identity with a pseudonym. Most jurisdictions do not allow that, and in any case your identity is revealed by other means.

^return

\subtitle{Subtitle of Book}

Your book does not require a subtitle. They are rarely used with popular fiction, but are widely used for non-fiction as a marketing tool.

If you use a subtitle, it must follow the rules for title, as described above. It must be identical, wherever the subtitle is written (cover, title page, Copyright page). But normally you do not put the subtitle in running page headers.

The subtitle does not appear in PDF metadata. However, it can be searched in the book distribution database.

^return

\mode{choice}

There are three modes: draft shade final. Default is draft.

  • The draft mode typesets paragraphs, and numbers pages, identical to how they will be typeset in final mode. So, you should do most of your writing in draft mode, until the final editing stage.
  • In draft mode, you may use "fake" images. These are placeholders, described here.
  • If you use real images, they will appear in draft mode, even if they are not eligible for final mode.

  • In draft mode, the internal PDF metadata will have DRAFT prepended to the book title. This does not change the printed title.

  • The shade mode is also draft mode, but with some areas shaded gray, so you can visualize alignment. This is illustrated here.
  • In final mode. Novelette performs several additional checks, which consume additional time (especially if you use images). Final mode is required for the PDF that you submit to the printer.

^return

\intent{choice}

There are five intents: none SWOP FOGRA39 JC200103 custom. Default is SWOP, which is an abbreviation for "U.S. Web Coated (SWOP) v2", the USA standard. FOGRA39 (acutally FOGRA39L) is one of several European standards. JC200103 is used in Japan.

  • If you set intent as none, then the resulting PDF will not declare that it conforms to PDF/X. It is still a good PDF, just not PDF/X.
  • Any other intent will write internal metadata, declaring that the PDF conforms to PDF/X-1a:2001 standards. This is widely accepted (and sometimes required) by print-on-demand services.
  • In any case, a color profile (*.icc file) will NOT be attached.
  • Do not use a custom intent, unless you are required to use a particular intent that is not one of the choices. Then, then you will need to write more settings in Preamble, according to industry specifications.
  • Dirty little secret: Usually, the intent is ignored by the printer. But since it is part of the PDF/X specification, you must have it, unless you do not need PDF/X. It is only important in commercial color offset printing, which is not the technology that will be used for your book.

^return

\booksize{w=width,h=height}

You must set booksize. There is no default. The setting requires both width and height, measured in units of inches (in) or millimeters (mm) or centimeters (cm).

Book size is known as trim size. This is because larger paper is fed into the print machine, then trimmed (cut) to actual size, during binding.

  • Your print service will offer you a choice of several standard trim sizes. In the USA, w=5.5in,h=8.5in is a popular size for softcover fiction. In the UK, perhaps B-format at w=129mm,h=198mm would be your choice.
  • Be sure to select off-white (cream, creme) paper. Not white.
  • Unless you know what you are doing, avoid the small "mass market" and "Pocket" sizes. Those are generally used only by commercial publishers with "instant best-seller" books (many of which will be discarded, at a loss).
  • Unless you know what you are doing, do not choose "expanded distribution" (or similar term) when you sign with the print service. You do not want booksellers to order copies (at your expense) in hopes they may sell, with your loss if they do not sell.

^return

\sheetfeed{choice}

Do not use the sheetfeed setting, unless you need it. Incorrect use of this setting may cause the printer to reject your PDF, or print it at the wrong size.

Choices: booksize letter A4. Default booksize.

  • When sheetfeed is booksize (default), the PDF page size is identical to the booksize setting. This is how most print services want it.
  • When the sheetfeed is letter or A4, the pages are centered (vertically and horizontlly) in the larger PDF paper size. This may be useful if you are printing a private copy on your home printer, for manual editing. A few print services will request sheetfeed, if there is a human editor.
  • Caution: If you use sheetfeed only for personal printing, be sure to un-set it before creating the final PDF.
  • Whether or not you use sheetfeed, the PDF will have internal metadata defining its "TrimBox", where the trimmed pages are located.

^return

\margins{t=T,o=O,b=B,i=I}

T, O, B, and I are measured in inches (in) or millimeters (mm). T is the top margin, O is the outside margin, B is the bottom margin, and I is the inside (spine) margin.

Default: t=13mm,o=13mm,b=13mm,i=16.2mm. This complies with the minimum values requested by many (not all) print services in the USA and Europe. Also, they look good, unles your booksize is very large or very small.

  • Normally, I is larger than O, because I must include the area that will be hidden from view when the book is bound (glued).
  • Margins are the white space surrounding the printable page area. In other words, nothing is placed in the margins. If you layout has header and/or footer, they will be printed in the printable rectangle surrounded by the empty margins. This method differs from some word processors, which place header/footer within the top/bottom margin areas.
  • There is no separate setting for gutter.

^return

\typesize{size}

The typesize is the font size of main body text. It is measured in units of TeX points (pt) or PostScript points (bp).

If you do not set typesize, Novelette will set it for you. This is often best. After reading other settings, Novelette will calculate a default typesize from 11pt-12bp, based on certain reading standards.

  • The typesize is a global setting. You may not change it later in the book. However, there are some locations (such as chapter names) where the text is scaled to another size.
  • A typical default calculates typesize near 11.4pt.
  • You cannot set the leading, known to TeX as "baselineskip". It will be calculated from other settings.

^return

\lines{integer}

This sets the number of lines per page, in main body text. It does not count the header and/or footer.

  • You do not need to set lines. After reading other settings, Novelette will calculate a default number of lines, typical for popular fiction. The result is likely to be in the range 28-35.
  • This is a global setting. You cannot change it later in the book. However, special display pages may disregard the line grid.

^return

\layout{style=S,hgap=H,fgap=F,num=N}

This sets the page layout. You may (and will) locally over-ride the style, for individual pages. Otherwise, it is a global setting.

Default: style=2,hgap=.5,fgap=.5,num=m. This creates a layout that has header and footer, with centered header text, centered page number in the footer, and page numbers using the same type style as main text.

There are many ways to combine the four layout settings. Here is a gallery of some possible combinations.

The style sets the header/footer style number:

  • style=1 provides a footer, but no header. Page numbers are centered in the footer.
  • style=2 provides header and footer. Running header text is centered. Page number is centered in the footer.
  • style=3 provides header and footer. Running header text aligns to outside margin. Page number, in footer, aligns to outside margin. This is a very unusual style.
  • style=4 provides only header. Running header text is centered. Page number is at outside margin.
  • style=5 is like style 4, with added vertical bar at a fixed distance from the margin. This style is widely-used.
  • style=6 is like style 4, with added slash / separated from the page number by a space. This style is frequently seen in technical works, but rarely in popular fiction.
  • style=7 provides only header. Both running text and page number are centered. They are separated by a spaced slash / with the page number twoard the outside. This style may be useful with smaller book sizes, because the header is less likely to crowd main text.
  • The above choices are the only styles that regularly appear in popular fiction. If you need something else, such as underlined header or double-line header, then do not use Novelette.
  • style=0 exists only for developer use. It has no header or footer, and no page numbers.
  • You cannot place images in header or footer.

The hgap is a number from 0 to 1, usually decimal. It is the amount of additional vertical space (measured as multiple of ordinary lines) between header and main body text.

  • hgap=0 can be set, but it looks bad. Avoid it.
  • hgap=1 provides a full empty line between header and main body text. Unless your booksize is large, hgap=1 might be too large.
  • hgap=.5 is the default. It looks good with common book sizes.
  • If the layout style does not use a header, then hgap is ignored.

The fgap is like hgap, except it appliesbetween main text and footer.

The num sets how page numbers will appear.

  • num=m uses the same page numerals as in main text. This is the default.
  • num=l uses lining numerals, if the font has them.
  • num=o uses oldstyle numerals, if the font has them.
  • You may add i to any of the above codes. This will change from default Regular, to Italic.
  • The num setting does not determine whether page numbers are lowercase roman letters, or arabic numerals. That will depend on whether the pages are in frontmatter, or mainmatter.
  • If you have frontmatter pages that display the page number (unusual), they will respond to the i setting.

^return

\guides{up to four comma-separated integers}

Guides are little gray rectangles, visible in the side margins, only when \mode{shade} is set. The use of guides is optional. Default is no guides.

  • You may set up to four guide pairs.
  • Format: \guides{J,K,L,M} where J, K, L, and M are integers (if you use all four guide pairs).
  • The integer is the line number of main body text, at which the guide will appear. Header and footer are ignored. So, a guide at 5 sets the markers aligned with line 5 from the top of main text.
  • The bottom of the guide rectangle is at the baseline of main text. The top of the guide rectangle is at the x-height of normal main text.
  • Guides are global. They are set the same on every page.
  • The purpose of guides is to provide a visual reference, particularly at chapter openings. There, the chapter name (and possibly description) will be placed at a certain number of lines from the top, and main text will begin lower down the page. Align them with the guides you set.
  • Guides do not force alignment. They are only a visual reference.
  • Display pages, such as the book title page, will often not be aligned to the guides.

^return

\typeface{Font Family}[OpenType Features]

The "typeface" is also known as the "main font". It is used in main body text, and is also used anywhere you do not locally select a different font.

This command has identical effect to the \setmainfont command, of the fontspec package. You may use either command.

  • The Font Family might be something such as "Adobe Garamond Pro" or "Libertinus Serif" or "Clara" or "Crimson Pro Regular". It does not need to be a LaTeX font. You may use any OpenType or TrueType font, as long as it is licensed for commercial usage.
  • It is your responsibility to determine whether the chosen font is properly licensed (free or paid) for your purpose. Do not use GNU Public License fonts, unless their license comes with an explicit Font Exception Clause.
  • The default typeface is "Swainson", which comes with Novelette. If you use Swainson, then the "Towhee" font will be used for footnotes.
  • Some fonts, such as "Latin Modern Roman", are not designed for the purpose of popular fiction. They do not offer the same flowing readability as fonts designed for the purpose.
  • The font family does not need Bold or BoldItalic. These are never used in the main text of popular fiction.
  • Some OpenType features are pre-selected: kern,liga,onum. That is, kerning, common ligatures, oldstyle numerals. You do not need to write these features.
  • When the font is used in some locations, such as page headers, openings, and display pages, the ligatures are not used. This is because such text is often loose (greater space) or enlarged.
  • The TeX-specific "TeX Ligatures" are also pre-selected.
  • Additional information regarding Opentype Features is on another page.

^return

\sansfont{Font Family}[OpenType Features]

This command has identical effect to the \setsansfont command, of the fontspec package. You may use either command.

  • The default sansfont is Swainson (which is serif), same as typeface.
  • In principle, this font should be be one of the sans-serif fonts. In practice, you may set any font, including those that have serifs.
  • The sans font might be useful in special locations, such as chapter openings, and display pages.
  • Also see the above information for typeface.

^return

\monofont{Font Family}[OpenType Features]

This command has identical effect to the \setmonofont command, of the fontspec package. You may use either command.

  • You probably will not need to use monospaced text.
  • The default monofont is "Latin Modern Mono", which comes with LaTeX.
  • Unlike other fonts, the monofont default features do not include kerning, ligatures, or oldstyle numbers.

  • A monospaced font is normally used for mathematics and tables, which are outside the purpose of Novelette. But if you need a monospace font, you may use it for special purposes.
  • Some monospace fonts are designed to look like traditional typing, from a mechanical typewriters. Others have a more modern design, and are used by computer programmers.
  • Also see the above information for typeface.

^return

\altfont{Font Family}[OpenType Features]

This command is specific to Novelette. It sets an "alternate font" that you can locally use, when you need characters that are not in the main font.

  • The default altfont is Swainson, same as typeface. But this is not helpful. You should select your own alternate font.
  • A good general-purpose altfont is "Libertinus Serif". This font has a large character set, and a design style that is inconspicious (usually) when mingled with a different main font.
  • You can install Libertinus Serif via your TeX package manager. It is package "libertinus-fonts" (not any other package with similar name). You will still need to set \altfont.
  • Use of the altfont is not automatic. You must manually request it where you need it, in the text: \alt{This text uses altfont}.
  • Also see the above information for typeface.

^return

\dramafont{Font Family}[OpenType Features]

This command is specific to Novelette. It sets a "dramatic font" that you can locally use, for special effect. That might be on the book title page, or in chapter openings.

  • You do not necessarily need a dramatic font.
  • The default dramafont is Swainson, same as typeface.
  • To use it: \drama{This is dramatic text}.
  • Also see the above information for typeface.

^return

\decofont{font-filename.otf}[OpenType Features]

This command is specific to Novelette. It sets a "decorative font" that you can locally use, for dingabts or other decorative elements.

Unlike the other font settings, decofont does not select a font family. It selects a single font file, which you choose by its full file name, with extension.

  • You do not necessarily need a decofont.
  • The default decofont is Swainson, same as typeface. Note that Swainson does not have any decorative characters.
  • Unlike other fonts, the decofont default features do not include kerning, ligatures, or oldstyle numbers.
  • To use it: \deco{character}.
  • Also see the above information for typeface.

^return

\namestyle{align=A,font=F,scale=S}

The \namestyle setting provides a uniform appearance for text that uses the \name command. Example here.

  • align=code determines the horizontal alignment. The code is one of l c r o i (left, center, right, outside inside). Default align=c.
  • font=code determines which font will be used for the name text. The code is one of main sans mono alt drama deco. Default font=main.
  • scale=number sets the scale of name text, relative to main font size. Default scale=1.6, which looks nice in chapter names.

^return

\descstyle{align=A,font=F,scale=S}

The \descstyle setting provides a uniform appearance for text that uses the \desc command. Example here.

The use of \descstyle is like that of \namestyle, except that this default scale=1.25.

^return

\setmicrotype{settings}

Novelette uses the microtype package for improved appearance of body text. Since the package does not load until after Preamble, this command provides a way to "PassOptionsToPackage" when microtype loads.

If you do not know what that means, simply ignore it. Novelette is pre-set to values that are good for its purpose. You do not need this command.

If you do wish to play with this command, Novelette will not let you use the microtype draft mode. Microtype will always be in its own final mode, no matter what you choose via \mode in Novelette.

^return

\myparindent{size}

Novelette pre-sets the default paragraph indent at 1.5em. That looks good for the purpose intended. But if you need a different default value, use \myparindent.

  • Example: \myparindent{10pt} for a small indent.
  • This is a global setting. If you need to locally change the indent, that is performed using other commands in body text.

^return

\warnpercent{on or off}

TeX uses the percent character % for comments. Anything following %, on the same line of raw text (prior to typesetting) will be ignored.

If you need to print the percent character, you may use a backslash \% there.

Default \warnpercent{off} does nothing. But \warnpercent{on} issues a Warning if you use the un-escaped % anywhere in body text. It does not warn for Preamble. This is a way to detect accidental use of % (which means comment) when you really meant to print %.

In body text, you may use \memo{This does not print.} where you need it. The difference between memo and %, is that after the memo completes, anything following it will print.

If you use this to issue a Warning, then the PDF will not be in final mode.

The novelette-docpages directory has a file named "warnmebro.tex". You may edit it with warnpercent on or off, for comparison.

^return

\warnquotes{on or off}

TeX normally converts straight double quotes " to right curly quotes ”. Alas, some writers (moi) are in the habit of typing " with the expectation that a smart-quotes feature will alternately convert them to left and right curly quotes, as needed. That is not what TeX does, so the print will be wrong. This is often hard to notice.

Default is \warnquotes{off}. If you use \warnquotes{on} then Novelette will issue a Warning if you wrote " anywhere in the document body. It does not warn for Preamble.

Since Novelette uses UTF-8 text encoding, you may directly paste left and right curly quotes from a character map. Or, you may use `` (two acute characters, also known as backticks) for left curly quotes, and two '' (straight single quotes, also known as straight apostrophe) for right curly quotes.

If you use this to issue a Warning, then the PDF will not be in final mode.

This documentation directory has a file named "warnmebro.tex". You may edit it with warnquotes on or off, for comparison.

^return

\warnalert{on or off}

Novelette sometimes writes ALERT messages to the log file. These are not Errors, or Warnings. Usually, an ALERT means that you set something to an unusual value. Did you really mean that, or not?

Default \warnalert{off}. If you use \warnalert{on} then a Warning will be issued if there were any Novelette-caused ALERTs.

It may be that the ALERT was thrown by a situation that you should fix. Or maybe it is what you intended, so no need to fix it. When you are satisfied that nothing needs to be fixed, turn \warnalert{off}.

If you use this to issue a Warning, then the PDF will not be in final mode.

This documentation directory has a file named "warnmebro.tex". You may edit it with warnalert on or off, for comparison.

^return

\application{string} \producer{string}

PDF metadata includes "application" and "producer". The "application" is the software that created the text (perhaps a word processor), and the "producer" is the application that converted the text to PDF (perhaps a Distiller). Modern software does both at once.

Novelette is pre-set to use "Novelette" as application, and as producer. Normally, you never need to change this. But if your PDF will be sent through a workflow that expects a specific string for either application or producer, you may use these commands to change the metadata.

^return
