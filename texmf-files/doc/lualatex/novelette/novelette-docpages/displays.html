Display Pages

A display page stands by itself. If you put too much on a display page, it will continue to a second display page, but this is discouraged. Best practice is to limit content, and use separate display pages.

  • Most or all pages in frontmatter are display pages.
  • A blank page simply uses \blankpage.
  • If your story is divided into isolated parts, then a display page may be used as separator.
  • If you add "About the Author" or something similar after then end of the story, it might be a display page.
  • Display pages do NOT appear within main text. If you need a full-page image, such as a map, use the \image command with option [p].

The display environment

\begin{display} starts a new page. \end{display} ends the page, so that following centent is on a new page.

A display page has empty header, empty footer. If you need to change this, use \thispagestyle. It applies only to the one display.

Within the display, and only within the display, you may use the \style command.

Since the display does not need to be on-grid, the \vspace*{distance} command may be used to create vertical space between elements. A negative distance moves them closer than they would ordinarily be. Warning: Do not use this command, with or without asterisk, anywhere except on display pages.

If you wish to move content to the lowest page position, use \vfill between \begin{display} and the start of its content. This is sometimes seen on copyright pages.

If you wish to vertically center content, write \vfil between \begin{document} and the content, and write \vfil after the content but before \end{display}. Note the single l in \vfil.

Look at the novelette-demo.tex file, to see how this is done.

\begin{display}
\style[options]{Some Text}
\style[options]{More Text}
\image[options]{filename.png}
\style[options]{and so forth}
\end{display}

The display environment starts a new page. If the layout has a header, it will be empty. When the display ends, whatever follows will be on another new page.

The \style command allows you to change the font, size, and alignment. Within the style, text may wrap to more than one line.

On a display page, lines are usually off-grid. This is not a problem, because main text is not used there.

You may also use images.

\style[align=A,font=F,scale=S]{styled text}

The \style command may only be used within a display or within an opening. Its options allow you to change the alignment, font, and scale of text. The styled text may wrap to more than one line. However, if there is more than one paragraph, use a separate \style command for each paragraph.

This command may be used as \style or \style*. Without the star (asterisk), multi-line text is printed with an integral number of line skips, at least as great as the scale. Example: At scale 2.1, lines will be triple-spaced.

With the star, \scale* squeezes the vertical skip between consecutive lines, so that is is just enough. This means that the text will no longer be on-grid, but it will look better. Since text does not need to be on-grid in a display page, the use of the starred command is best. But since text must obey line grid in openings, the star will be silently ignored there.

  • This is a local command. It applies only to its own styled text.
  • You may set all, or some, of the options. Anything unset will use the default.
  • A is one of l c r o i for left, center, right, outside, or inside alignment. Default center.
  • F is one of main sans mono alt drama deco. It chooses the font. Default main.
  • S is a number, which scales relative to main text. For example, scale=1.8 will scale 1.8x main text. Default 1.
  • Within the {styled text} you may use additional styling commands such as \ital{text} for Italics, \smcp{text} for small caps, and others. You may also use \bold{text}, which is not available in main text.
  • If you need something fancy, such as rotated or distorted text, then you must create it as an image using other software. In any case, it must be black and white raster only.
