Miscellaneous

\par   \\   \br   \linebreak

In the plain-text *.tex files, a paragraph is ended when its text is followed by an empty line. You may use \par to end a paragraph, without needing to use an empty line. Multiple empty lines, and/or multiple \par, do not create additional paragraphs or line spaces.

To break a line mid-paragraph, you may use \\ or \br there. The broken line will not extend to full text width.

\linebreak may be used to break a line mid-paragraph, but in this case the broken line is justified to full width. In most cases, the added spacing will look ugly, and you will need to re-write the paragraph.

Do not use any kind of line break to end paragraphs. Only use a line break when you cannot complete a paragraph there. For example, you may break lines within \centering, but you may only use \par at its end.

\blankpage

This does what it says. Nothing prints there. If issued more than once, several blank pages are created.

\thispagestyle{setting}

Novelette automatically prints page header and/or footer in some locations, but not others. In frontmatter, no header/footer is printed. In mainmatter, pages using upperpage or fullpage will not print header (but will print footer, if layout has it). Otherwise, mainmatter pages print header/footer.

Command \thispagestyle will locally change that behavior. The change applies only to the one page where the command is written. If you need to apply it to multiple pages, then you must re-use the command somewhere on each page (does not need to be at the top).

  • \thispagestyle{empty} : No header or footer prints.
  • \thispagestyle{main} : Header/footer print, according to whatever was set by \layout in Preamble.
  • \thispagestyle{footer} : No header prints. If the layout has a footer, it prints. But if the layout does not have a footer, this command does not create one, so nothing prints there.
  • \thispagestyle{plain} : Same as {footer}.
\forceindent and \backindent

Paragraphs are automatically indented, or not indented, according to context. These two commands apply a local change.

  • \forceindent applies a standard paragraph indent. You may use several of them, for multiple indent.
  • \backindent shifts text backwards, so that it over-writes an existing indent. Caution: If you use this command where there was no indent, or if you use it more than once, then the text may shift into the margin. This is an error, which Novelette cannot detect.
{\centering Text to be centered.\par}

This produces one or more lines of centered text. You cannot have more than one paragraph, but you may use \\ to break lines. You may use an \icon.

In most cases, it is better to use the center environment.

\clearpage

Finishes the page. Following content starts on a new page. Does not create a blank page. If issued more than once, only a single \clearpage is obeyed, and the others are silently ignored.

\cleartorecto

Like \clearpage, but if the page ends recto (odd number), then a blank page is automatically inserted, so that the following content starts on a recto page. This command is automatically used within the \mainmatter and \backmatter commands.

Do not issue this command more than once in succession.

\null

Creates a blank line. May be repeated for several blank lines. This command may only be used between complete paragraphs, and/or \image. Although this command may be used anywhere, it is best to use it only within upperpage and on display pages.

\vfil and \vfill

These commands should only be used on display pages. If you use them anywhere else, they will throw lines off-grid.

\vfill places "all remaining vertical space" there. On the book title page, you may place it between title/author and the publisher's line, to force the publisher's line to the bottom of the page. On the copyright page, placing it above the other content will force all content downward.

\vfil (just one l) may be used multiple times on a page. Remaining vertical space will be equally divied among them. If you place \vfil above and below other content, then the content will be vertically centered.

\ital{Text in Italics font.}

This is an alias for the LaTeX \textit command. If used within a paragraph, there will be a small amount of space added after the final character, to allow for slant. You may enclose more than one full or partial paragraph. Also see the \emph command.

If you are using styled text with font=main (the default), then Italics are available. But they are not available when the font is light or dark.

\emph{Emphasized text, Italics or not.}

This is a toggle between Regular and Italics. If surrounding text is Regular, then \emph provides Italics. If the surrounding text is Italics, then \emph provides Regular. You may nest the command.

\small, \large, and similar commands do not work

Surprise! LaTeX size commands do not work in Novelette. This is intentional. Popular fiction does not change its text size, except in special locations. If you use any of these commands, the text will be printed without size change.

Using different commands, size changes are allowed in the upperpage environment and on display pages.

Short-Form Feature Commands

Some OpenType features are often added or removed. For your convenience, here are short forms. None of these affect features such as style sets ssNN or character variants cvNN.

  • \onum{text} removes any other number style, then adds onum. It does not affect letters.
  • \lnum{text} removes any other number style, then adds lnum. It does not affect letters.
  • \smcp{text} removes any other letter style, then adds smcp. It does not affect numbers or uppercase letters.
  • \allsmcp{text} combines smcp and c2sc and onum.

\kern and \sk

TeX provides \kernvalue for tweaking the distance between consecutive letters:

class="code">T\kern-.012emo be, or not to be.

Unfortunately, that is a bit hard to read. You may use it, or use the Novelette \sk{} command (stemwidth kern). The braces take a number (without units). A stemwidth is the average width of vertical stems in letters, which is about .08em. This makes it easy to judge how much kerning is needed, since you have a visual guide. The above code could be written:

T\sk{-1.5}o be, or not to be.
